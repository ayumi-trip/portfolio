S3 + CloudFront で静的サイトをホスティングする方法

AWS S3 と CloudFront を組み合わせると、スケーラブルで低コストな静的サイトのホスティング環境を簡単に構築できます。 この記事では、バケット作成からカスタムドメインの設定まで、一連の手順を解説します。

1. S3 バケットの作成

まず S3 バケットを作成します。バケット名はドメイン名と一致させると管理がしやすくなります。 パブリックアクセスは後述の CloudFront 経由のみ許可するため、この段階ではブロックしたままにします。 

aws s3 mb s3://your-domain.com --region ap-northeast-1

2. 静的ウェブサイトホスティングを有効化

バケットのプロパティから「静的ウェブサイトホスティング」を有効にし、 インデックスドキュメントに index.html、エラードキュメントに 404.html を指定します。

3. CloudFront ディストリビューションの作成

CloudFront コンソールから新しいディストリビューションを作成し、 オリジンドメインに先ほど作成した S3 バケットを指定します。 OAC（Origin Access Control）を使うことで、S3 バケットへの直接アクセスを防ぎ、 CloudFront 経由のみでコンテンツを配信できます。

OAC は旧来の OAI（Origin Access Identity）の後継で、よりセキュアな設定が可能です。

4. カスタムドメインと SSL 証明書の設定

Route 53 でドメインを管理している場合、ACM（AWS Certificate Manager）で SSL 証明書を発行し、 CloudFront の「代替ドメイン名 (CNAME)」に追加するだけで HTTPS 対応が完了します。

  • ACM でリクエスト → DNS 検証
  • CloudFront のディストリビューション設定に証明書を紐付け
  • Route 53 で A レコード（エイリアス）を CloudFront に向ける

5. デプロイの自動化

ファイルを更新するたびに手動でアップロードするのは非効率です。 GitHub Actions などの CI/CD ツールを使って、main ブランチへのプッシュで自動デプロイする仕組みを作りましょう。 

aws s3 sync ./dist s3://your-domain.com --delete
aws cloudfront create-invalidation \
  --distribution-id XXXXXXXXX \
  --paths "/*"

以上でサイトが公開されます。設定が完了したら CloudFront の URL でアクセスして動作確認してみてください。
